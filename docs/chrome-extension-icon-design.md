# Knovana Chrome Extension Icon 设计理念

## 1. 核心概念

Knovana 的 Chrome 扩展图标采用 **书籍 + 书签 + K monogram** 的组合隐喻。

图标不应只是把字母 `K` 放进圆角方块里，而应让 `K` 从知识管理场景中自然生成：

- **书籍** 表达知识库、沉淀、长期保存和可回看。
- **书签** 表达浏览器场景下的采集、标记、收藏和快速进入工作流。
- **K** 保留 Knovana 的品牌识别，同时呼应 Knowledge。

当前方向定义为：

> 一个由书脊和书签折页共同构成的 K。左侧是知识库主脊，右侧是从书中伸出的书签 ribbon，上下展开后形成 K 的两翼。

## 2. 视觉结构

图标使用透明背景，只保留由书脊和书签折面组成的 K 本体，避免额外圆角底板或外围边框干扰工具栏中的识别。

主要结构：

- **透明画布**：不使用额外底板、描边或外框，让 K 本身成为唯一视觉主体。
- **左侧蓝色竖形**：当前承担书脊和知识库主册的角色。
- **右侧蓝绿 ribbon**：表达书签、采集入口和知识流动。
- **中心折叠点**：不再使用白色空洞，而是使用有色折叠结构连接书脊和书签，使图形更完整。

配色原则：

- 蓝色用于表达可靠、工具、采集和结构化。
- 青绿色用于表达知识沉淀、连接和可复用。
- 透明背景用于减少工具栏中的卡片感；小尺寸清晰度主要依赖 K 本体的高对比色块。

## 3. 小尺寸策略

Chrome 扩展图标需要同时服务 `16/32/48/96/128px` 尺寸。

适配原则：

- `128px` 和 `96px` 保留书脊、书签折叠和轻微高光。
- `48px` 保留完整主轮廓，减少细节。
- `32px` 只保留可读的 K 形结构和主要色块。
- `16px` 以识别度优先，不强行展示折叠细节。

不要在小尺寸中保留复杂负形、细线节点或过多高光，否则会产生脏点和误读。

## 4. 当前不足

当前版本已经比纯字母 K 更贴近知识库语义，但左侧蓝色竖形仍然偏普通圆角色块，**书脊感还不够明确**。

主要问题：

- 左侧缺少书籍侧边的厚度、页边或封面折线暗示。
- 竖形过于规整，容易被看成普通 K 的竖笔，而不是书脊。
- 书签从书中伸出的关系还可以更自然，当前连接点仍偏几何化。

## 5. 后续优化方向

下一轮图标优化建议围绕 **增强书脊识别** 展开，而不是重新引入抽象负形。

可选优化：

- 将左侧竖形改成更像一本窄书的侧面：保留圆角，但加入轻微内侧页边线。
- 在左侧顶部或底部加入非常克制的书页切面，使其不像普通矩形。
- 让右侧书签从书脊背后自然伸出，形成前后层次。
- 保持整体 K 的轮廓稳定，避免为表达书籍而牺牲工具栏识别度。

需要避免：

- 不再使用中间白色空洞表达洞察，因为实际视觉上容易产生困惑。
- 不使用通用节点网络图形，避免落入知识图谱图标俗套。
- 不模仿 Gemini 星形标识，仅借鉴白底中心符号的小尺寸可读性。

## 6. 资产约定

当前图标资产位于：

```text
chrome-extension/public/icon/source.svg
chrome-extension/public/icon/16.png
chrome-extension/public/icon/32.png
chrome-extension/public/icon/48.png
chrome-extension/public/icon/96.png
chrome-extension/public/icon/128.png
```

`source.svg` 是设计源文件。更新图标时应先调整 `source.svg`，再重新生成各尺寸 PNG，避免 PNG 与设计源不一致。
